Review

Git is a Distributed Version Control System. It allows teammates to work on the same files and have a central place to store those files. Teammates can pull from the DVCS, make their changes and then push them back to the central storage place. This makes making the sites easy to differentiate and, as long as teammates store the most recent files, easy to update.

As linked on the home page there is a list of git commands that should be reviewed. That link is here.

However the more difficult ones are here. They are as follows:

Unstaging a commit

git reset HEAD (filename).txt

Skip staging and Commit

git commit -a -m "command"

Adding to a Previous Commit

git commit --ammend -m "command"

Pulling Last Commit

git reset --soft HEAD^

Skipping Staging and Commit

git commit -a -m "modify"

Creating a new branch

git branch (name)

Create branch and checkout

git checkout -b (branch-name)

Deleting remote branch

git push origin :(filename)

Clean up remote branches

git remote prune (origin or other name)

Rebase

This is a type of merge and merge commit. This may help.

git fetch and git rebase
